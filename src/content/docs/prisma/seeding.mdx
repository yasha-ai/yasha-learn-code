Сидирование (seeding) — это процесс наполнения базы данных начальными данными. Prisma поддерживает официальный механизм сидирования через файл `prisma/seed.ts`.

## Настройка seed скрипта

Добавьте скрипт в `package.json`:

```json
{
  "prisma": {
    "seed": "ts-node prisma/seed.ts"
  }
}
```

Установите зависимости:

```bash
npm install -D ts-node
npm install @faker-js/faker  # для генерации фейковых данных
```

## Базовый seed файл

```typescript
// prisma/seed.ts
import { PrismaClient } from '@prisma/client'

const prisma = new PrismaClient()

async function main() {
  // Создать роли
  await prisma.user.createMany({
    data: [
      { email: 'admin@example.com', name: 'Администратор', role: 'ADMIN' },
      { email: 'user@example.com', name: 'Тестовый пользователь', role: 'USER' },
    ],
    skipDuplicates: true,
  })
}

main()
  .catch(console.error)
  .finally(() => prisma.$disconnect())
```

## Запуск сидирования

```bash
npx prisma db seed
```

## Использование Faker.js

```typescript
import { faker } from '@faker-js/faker/locale/ru'

const users = Array.from({ length: 50 }, () => ({
  email: faker.internet.email(),
  name: faker.person.fullName(),
  bio: faker.lorem.sentence(),
}))

await prisma.user.createMany({ data: users, skipDuplicates: true })
```

## Условное сидирование

```typescript
// Создать только если таблица пустая
const count = await prisma.user.count()
if (count === 0) {
  await seedUsers()
}
```
