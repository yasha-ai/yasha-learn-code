Atomicity, Consistency, Isolation, Durability.

## ACID свойства

### Atomicity (Атомарность)

Всё или ничего.

```sql
BEGIN;
UPDATE accounts SET balance = balance - 100 WHERE id = 1;
UPDATE accounts SET balance = balance + 100 WHERE id = 2;
COMMIT;  -- Обе операции или ни одна
```

### Consistency (Консистентность)

Данные всегда валидны.

```sql
ALTER TABLE accounts ADD CONSTRAINT positive_balance CHECK (balance >= 0);

-- Транзакция откатится, если баланс станет отрицательным
```

### Isolation (Изоляция)

Транзакции не влияют друг на друга.

```sql
-- Transaction 1
BEGIN;
UPDATE accounts SET balance = balance - 100 WHERE id = 1;
-- ещё не COMMIT

-- Transaction 2 не видит изменений Transaction 1
SELECT balance FROM accounts WHERE id = 1;  -- старое значение
```

### Durability (Долговечность)

После COMMIT данные сохранены навсегда (даже при сбое).

## Уровни изоляции

```sql
-- READ UNCOMMITTED (самый слабый)
SET TRANSACTION ISOLATION LEVEL READ UNCOMMITTED;

-- READ COMMITTED (по умолчанию в PostgreSQL)
SET TRANSACTION ISOLATION LEVEL READ COMMITTED;

-- REPEATABLE READ
SET TRANSACTION ISOLATION LEVEL REPEATABLE READ;

-- SERIALIZABLE (самый строгий)
SET TRANSACTION ISOLATION LEVEL SERIALIZABLE;
```

// A - Atomicity (атомарность): всё или ничего
let balances = { alice: 1000, boris: 500 };

function transfer(from, to, amount) {
  const backup = { ...balances };
  try {
    balances[from] -= amount;
    if (balances[from] < 0) throw new Error("Недостаточно средств");
    balances[to] += amount;
    console.log("Перевод " + amount + ": успех");
  } catch (e) {
    balances = backup; // ROLLBACK
    console.log("Перевод " + amount + ": откат -", e.message);
  }
}

transfer("alice", "boris", 300);  // OK
console.log("Балансы:", balances);

transfer("alice", "boris", 5000); // Откат!
console.log("Балансы после отката:", balances);

// I - Isolation (изоляция): транзакции не мешают друг другу

console.log("READ UNCOMMITTED — видны незакоммиченные данные");
console.log("READ COMMITTED — видны только закоммиченные");
console.log("REPEATABLE READ — повторное чтение стабильно");
console.log("SERIALIZABLE — полная изоляция");

// D - Durability: после commit данные сохранены
