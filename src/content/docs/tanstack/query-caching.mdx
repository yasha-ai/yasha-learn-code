## Как работает кэш TanStack Query

TanStack Query автоматически кэширует результаты запросов. Понимание жизненного цикла кэша —
ключ к правильной настройке производительности приложения.

## Состояния данных в кэше

Данные в кэше TanStack Query могут находиться в одном из состояний:

### Fresh (Свежие)
Данные только что получены и считаются актуальными. TanStack Query **не будет** делать новый запрос,
даже если компонент перемонтируется или окно получит фокус. Продолжительность — `staleTime`.

### Stale (Устаревшие)
Данные есть в кэше, но уже могли устареть. TanStack Query **автоматически обновит** их в фоне
при следующем обращении (фокус окна, монтирование компонента, переключение вкладок).

### Inactive (Неактивные)
Компоненты, использующие данные, размонтированы. Данные ещё хранятся в кэше в течение `gcTime`.

### Deleted (Удалённые)
Данные удалены из кэша после истечения `gcTime`. Следующий запрос будет выполнен с нуля.

## staleTime: Время до устаревания

```typescript
useQuery({
  queryKey: ['users'],
  queryFn: getUsers,
  staleTime: 1000 * 60 * 5,  // 5 минут
})
```

- По умолчанию: `0` (данные устаревают мгновенно после получения)
- При `staleTime: Infinity` данные никогда не устаревают автоматически
- Хорошие значения: 1-5 минут для часто меняющихся данных, `Infinity` для статичных (например, список стран)

## gcTime: Время жизни в кэше (Garbage Collection)

```typescript
useQuery({
  queryKey: ['users'],
  queryFn: getUsers,
  gcTime: 1000 * 60 * 10,  // 10 минут (было cacheTime в v3)
})
```

- По умолчанию: `5 минут`
- Отсчёт начинается когда запрос становится неактивным (нет подписчиков)
- `gcTime` должен быть >= `staleTime` (иначе данные удалятся раньше, чем устареют)

## Фоновые обновления

Когда данные устарели (stale), но ещё в кэше, TanStack Query показывает старые данные мгновенно
и **одновременно** делает фоновый запрос за новыми. Это обеспечивает моментальный отклик интерфейса.

```typescript
const { data, isFetching, isStale } = useQuery(...)
// data — старые данные (показываем сразу)
// isFetching === true — идёт фоновый запрос
// isStale === true — данные устарели
```

## refetchOnWindowFocus

По умолчанию TanStack Query обновляет устаревшие данные при фокусе вкладки браузера.
Имитирует поведение нативных приложений — данные всегда актуальны.

```typescript
// Отключить для всего приложения
const queryClient = new QueryClient({
  defaultOptions: { queries: { refetchOnWindowFocus: false } }
})

// Отключить для конкретного запроса
useQuery({ ..., refetchOnWindowFocus: false })
```

type CacheStatus = 'fresh' | 'stale' | 'inactive' | 'deleted'

function useCacheLifecycle(staleTime: number, gcTime: number) {
  const [status, setStatus] = useState<CacheStatus>('fresh')
  const [elapsed, setElapsed] = useState(0)
  const [subscribed, setSubscribed] = useState(true)
  const [fetchCount, setFetchCount] = useState(1)
  const [bgFetching, setBgFetching] = useState(false)
  const intervalRef = useRef<ReturnType<typeof setInterval> | null>(null)
  const startRef = useRef(Date.now())
  const inactiveRef = useRef<number | null>(null)

  useEffect(() => {
    setStatus('fresh')
    setElapsed(0)
    startRef.current = Date.now()
    inactiveRef.current = null

    intervalRef.current = setInterval(() => {
      const now = Date.now()
      const e = now - startRef.current
      setElapsed(e)

      if (!subscribed) {
        if (inactiveRef.current === null) inactiveRef.current = e
        const inactiveFor = e - (inactiveRef.current ?? e)
        if (inactiveFor >= gcTime) {
          setStatus('deleted')
          clearInterval(intervalRef.current!)
        } else {
          setStatus('inactive')
        }
      } else {
        inactiveRef.current = null
        if (e >= staleTime) {
          setStatus('stale')
        } else {
          setStatus('fresh')
        }
      }
    }, 100)

    return () => clearInterval(intervalRef.current!)
  }, [staleTime, gcTime, subscribed, fetchCount])

  const triggerBgFetch = () => {
    if (status !== 'stale') return
    setBgFetching(true)
    setTimeout(() => {
      setFetchCount(c => c + 1)
      startRef.current = Date.now()
      setStatus('fresh')
      setElapsed(0)
      setBgFetching(false)
    }, 800)
  }

  return { status, elapsed, subscribed, setSubscribed, fetchCount, bgFetching, triggerBgFetch }
}
