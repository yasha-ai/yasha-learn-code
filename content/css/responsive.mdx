## CSS: Responsive Design

![Иллюстрация к уроку](/lessons/css-responsive.png)


### Что такое Responsive Design?

Responsive Design (адаптивный дизайн) — это подход к веб-разработке, при котором веб-сайты и приложения динамически подстраиваются под размер экрана, ориентацию и возможности устройства пользователя. Цель — обеспечить оптимальный опыт просмотра и взаимодействия, независимо от того, использует ли пользователь смартфон, планшет, настольный компьютер или любой другой девайс.

Ключевые принципы:
1.  **Гибкие (Fluid) Сетки**: Использование относительных единиц (проценты, `fr`, `vw`, `vh`) вместо фиксированных пикселей.
2.  **Гибкие Изображения и Медиа**: Изображения и видео, которые масштабируются пропорционально.
3.  **Медиа-Запросы (Media Queries)**: CSS-правила, применяемые в зависимости от характеристик устройства.

### Основы Responsive Design

#### `<meta name="viewport">`

Это **обязательный** мета-тег, который необходимо добавить в `<head>` вашего HTML-документа. Он сообщает браузеру, как масштабировать страницу на мобильных устройствах.

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

*   `width=device-width`: Устанавливает ширину области просмотра (viewport) равной ширине экрана устройства.
*   `initial-scale=1.0`: Устанавливает начальный коэффициент масштабирования 1:1, предотвращая автоматическое масштабирование браузером.

#### Гибкие (Fluid) Сетки

Используйте относительные единицы для ширины и отступов, чтобы макеты автоматически адаптировались.

```css
.container {
  width: 90%; /* Занимает 90% ширины родителя */
  max-width: 1200px; /* Но не более 1200px */
  margin: 0 auto; /* Центрирование блока */
  padding: 2vw; /* Отступы зависят от ширины viewport */
}

.column {
  width: 30%; /* Три колонки по ~30% */
  margin-right: 2%; /* Отступ между колонками */
  box-sizing: border-box; /* Включаем padding и border в общую ширину */
}
```

#### Гибкие Изображения и Медиа

Изображения не должны выходить за пределы своих контейнеров.

```css
img, video {
  max-width: 100%; /* Максимальная ширина изображения равна 100% его родителя */
  height: auto;    /* Высота автоматически подстраивается, сохраняя пропорции */
  display: block;  /* Удаление лишнего пространства под изображением */
}
```

### Медиа-Запросы (Media Queries)

Медиа-запросы позволяют применять CSS-стили только при соблюдении определенных условий (например, при определенной ширине экрана).

```css
/* Базовые стили для всех устройств (Mobile First) */
body {
  font-size: 16px;
  background-color: lightblue;
}

/* Стили для экранов шириной от 768px и больше (для планшетов и десктопов) */
@media screen and (min-width: 768px) {
  body {
    font-size: 18px;
    background-color: lightgreen;
  }
  .navigation {
    display: flex; /* На больших экранах меню становится flex-контейнером */
    justify-content: space-around;
  }
}

/* Стили для экранов шириной от 1024px и больше (для десктопов) */
@media screen and (min-width: 1024px) {
  body {
    font-size: 20px;
    background-color: lightcoral;
  }
  .main-grid {
    display: grid; /* На десктопах используем CSS Grid */
    grid-template-columns: repeat(3, 1fr); /* 3 колонки */
    gap: 20px;
  }
}
```

### Современные Подходы и Инструменты

#### Flexbox для Адаптивных Компонентов

Flexbox идеально подходит для создания адаптивных навигационных меню, карточек и других компонентов, где элементы должны выстраиваться в ряд или переноситься.

```css
.card-container {
  display: flex;
  flex-wrap: wrap; /* Позволяет карточкам переноситься на новую строку */
  justify-content: center; /* Центрирование карточек */
  gap: 20px; /* Отступы между карточками */
}

.card {
  flex: 1 1 300px; /* flex-grow, flex-shrink, flex-basis: Гибкий размер, мин. 300px */
  background-color: white;
  padding: 20px;
  box-shadow: 0 2px 5px rgba(0,0,0,0.1);
  min-width: 280px; /* Гарантируем минимальную ширину */
}
```

#### CSS Grid для Адаптивных Макетов

CSS Grid предоставляет мощные инструменты для создания сложных, но гибких макетов. `repeat(auto-fit, minmax(...))` — это золотой стандарт адаптивного Grid.

```css
.gallery {
  display: grid;
  /* Автоматически создает колонки: минимум 200px, максимум 1fr, подстраиваясь под доступное пространство */
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
  gap: 15px;
  padding: 10px;
}

.gallery-item {
  border: 1px solid #ccc;
  padding: 10px;
  text-align: center;
}
```

#### CSS Custom Properties (Переменные) в Responsive Design

Использование CSS переменных позволяет легко изменять значения свойств в медиа-запросах, централизованно управляя стилями.

```css
:root {
  --base-font-size: 16px;
  --spacing: 1rem;
  --card-bg: #f9f9f9;
}

body {
  font-size: var(--base-font-size);
  padding: var(--spacing);
}

.card {
  background-color: var(--card-bg);
  margin-bottom: var(--spacing);
}

@media screen and (min-width: 768px) {
  :root {
    --base-font-size: 18px; /* Увеличиваем шрифт */
    --spacing: 1.5rem;   /* Увеличиваем отступы */
    --card-bg: #e0f2f7;  /* Меняем цвет карточек */
  }
}
```

### Типичные Ошибки

*   **Забывать `<meta name="viewport" ...>`**: Самая частая причина проблем на мобильных устройствах.
*   **Использование фиксированных значений**: `width: 960px;` вместо `max-width: 960px;` или `width: 90%;`.
*   **Слишком много медиа-запросов**: Иногда Flexbox (`flex-wrap`, `flex-basis`) или Grid (`auto-fit`, `minmax`) могут решить проблему адаптивности без дополнительных `@media` правил.
*   **Тестирование только на одной ширине**: Обязательно проверяйте свой дизайн на различных устройствах и в разных ориентациях (портрет/пейзаж) с помощью инструментов разработчика.

### 🎯 Практика

1.  Создайте HTML-файл с простой структурой: `<header>`, `<main>` с несколькими `<article>`-блоками (или карточками), `<footer>`.
2.  Примените стили CSS:
    *   Убедитесь, что `meta viewport` присутствует.
    *   Используйте `max-width: 100%; height: auto;` для изображений.
    *   Сделайте `<main>` контейнером с использованием CSS Grid, чтобы на маленьких экранах карточки отображались в одну колонку, на средних — в две, на больших — в три.
    *   Используйте CSS Custom Properties для изменения `gap` в Grid и `font-size` в заголовках через медиа-запросы.

### 💡 Совет

Всегда начинайте разработку с "Mobile First" подхода. Сначала создавайте стили для самых маленьких экранов, а затем, с помощью `min-width` медиа-запросов, постепенно добавляйте стили для больших экранов. Это упрощает управление стилями и часто приводит к более чистому и эффективному коду.
