## CSS: Анимации и Transitions

В мире веб-разработки динамические и интерактивные интерфейсы — это не роскошь, а ожидаемый стандарт. CSS-анимации и переходы (transitions) позволяют создавать эти интерактивные элементы, делая пользовательский опыт более приятным и интуитивно понятным.

### Введение: В чем разница?

Хотя и `transitions`, и `animations` используются для создания движения на веб-странице, они служат разным целям:

*   **Transitions (переходы)**: Позволяют плавно изменять значения CSS-свойств при смене их состояний. Например, когда кнопка меняет цвет при наведении курсора. Это реакция на событие (например, `:hover`, `:active`, изменение класса через JavaScript).
*   **Animations (анимации)**: Позволяют создавать более сложные, многошаговые последовательности движений, которые могут запускаться автоматически или по команде. Они определяются с помощью правил `@keyframes`, описывающих состояние элемента в различных точках временной шкалы анимации.

### Transitions: Плавные переходы состояний

CSS Transitions обеспечивают простой способ анимировать изменения свойств CSS. Они требуют начального и конечного состояния, и браузер автоматически интерполирует между ними.

Основные свойства `transition`:

*   `transition-property`: Указывает, какое свойство будет анимироваться (например, `background-color`, `transform`, `all`).
*   `transition-duration`: Определяет время, за которое переход должен завершиться (например, `0.3s`, `300ms`).
*   `transition-timing-function`: Задает кривую скорости перехода (например, `ease`, `linear`, `ease-in-out`, `cubic-bezier(...)`).
*   `transition-delay`: Устанавливает задержку перед началом перехода.

```css
/* Пример 1: Плавное изменение кнопки при наведении */
.button-transition {
  background-color: #3498db;
  color: white;
  padding: 10px 20px;
  border: none;
  border-radius: 5px;
  cursor: pointer;
  /* Указываем, какие свойства анимируем и за какое время */
  transition-property: background-color, transform;
  transition-duration: 0.3s, 0.2s; /* Разная длительность для разных свойств */
  transition-timing-function: ease-in-out;
}

.button-transition:hover {
  background-color: #2980b9;
  transform: scale(1.05); /* Увеличиваем размер */
}

/* Пример 2: Сокращенная запись */
.menu-item {
  opacity: 1;
  transform: translateX(0);
  transition: opacity 0.4s ease-out, transform 0.4s ease-out 0.1s; /* Задержка для transform */
}

.menu-item.hidden {
  opacity: 0;
  transform: translateX(-20px);
}
```

### Animations: Свобода движения с @keyframes

CSS Animations позволяют создавать сложные анимации, определяя последовательность стилей в различных точках (`keyframes`) анимации.

**Правило `@keyframes`**:
Это основной блок для определения анимации. Вы задаете имя анимации и описываете стили элемента в определенных "ключевых кадрах", используя процентное соотношение от `0%` (начало) до `100%` (конец) или ключевые слова `from` (эквивалентно `0%`) и `to` (эквивалентно `100%`).

```css
/* Определяем анимацию "pulse" */
@keyframes pulse {
  0% {
    transform: scale(1);
    background-color: #2ecc71;
  }
  50% {
    transform: scale(1.1);
    background-color: #27ae60;
  }
  100% {
    transform: scale(1);
    background-color: #2ecc71;
  }
}
```

**Свойства `animation`**:

*   `animation-name`: Имя анимации, определенное в `@keyframes`.
*   `animation-duration`: Длительность одного цикла анимации.
*   `animation-timing-function`: Кривая скорости анимации.
*   `animation-delay`: Задержка перед началом первого цикла.
*   `animation-iteration-count`: Сколько раз анимация должна повториться (`infinite` для бесконечного повторения).
*   `animation-direction`: Направление воспроизведения (`normal`, `reverse`, `alternate`, `alternate-reverse`).
*   `animation-fill-mode`: Стиль, который элемент принимает до или после выполнения анимации (`none`, `forwards`, `backwards`, `both`).
*   `animation-play-state`: Состояние воспроизведения (`running`, `paused`).

```css
/* Пример 3: Пульсирующая кнопка с @keyframes */
.animated-button {
  padding: 15px 30px;
  border: none;
  border-radius: 8px;
  cursor: pointer;
  color: white;
  /* Применяем анимацию "pulse" */
  animation-name: pulse;
  animation-duration: 2s;
  animation-iteration-count: infinite; /* Бесконечное повторение */
  animation-timing-function: ease-in-out;
}

/* Пример 4: Загрузочный спиннер */
@keyframes spin {
  from { transform: rotate(0deg); }
  to { transform: rotate(360deg); }
}

.spinner {
  width: 50px;
  height: 50px;
  border: 5px solid #f3f3f3;
  border-top: 5px solid #3498db;
  border-radius: 50%;
  animation: spin 1s linear infinite;
}

/* Пример 5: Использование animation-fill-mode */
@keyframes fadeInAndStay {
  0% { opacity: 0; }
  100% { opacity: 1; }
}

.fade-in-element {
  opacity: 0; /* Начальное состояние до анимации */
  animation: fadeInAndStay 1s ease-out 2s forwards; /* Анимация запускается через 2с и остается в конечном состоянии */
  /* forwards: элемент сохраняет стили из 100% кадра после завершения */
}
```

### Типичные ошибки

1.  **Забытый `transition-duration`**: Если вы указали `transition-property`, но не указали `transition-duration`, анимация не будет видна, так как она завершится мгновенно (длительность по умолчанию `0s`).
2.  **Неправильное использование `animation-fill-mode`**: Если элемент должен оставаться в конечном состоянии анимации, но этого не происходит, скорее всего, вы забыли `animation-fill-mode: forwards;`.
3.  **Анимация дорогих свойств**: Анимация свойств, таких как `width`, `height`, `left`, `top`, `margin`, `padding` (свойств, влияющих на разметку), может быть ресурсоемкой. Для лучшей производительности используйте `transform` (для изменения положения, размера, поворота) и `opacity`.
4.  **Слишком быстрые/медленные анимации**: Баланс важен. Анимации должны быть заметны, но не раздражать. Обычно 0.2s - 0.5s для transitions, 1s - 3s для более сложных animations.

### 💡 Совет

Для оптимизации производительности при работе с анимациями и переходами используйте свойство `will-change`. Оно подсказывает браузеру, какие свойства элемента будут анимироваться, позволяя ему заранее оптимизировать рендеринг. Используйте его осторожно и только для элементов, которые действительно будут анимироваться, иначе это может привести к снижению производительности.

```css
.animated-element {
  will-change: transform, opacity; /* Подсказываем браузеру */
  transition: transform 0.3s, opacity 0.3s;
}
```
