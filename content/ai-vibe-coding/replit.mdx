# Replit: Cloud IDE с AI агентом

Replit — это облачная среда разработки, которая работает прямо в браузере. Никаких установок, никакой настройки окружения. Открыл — и сразу кодишь.

В 2025-2026 году Replit стал чем-то большим: платформой с AI агентом, который пишет код, дебажит, тестирует и деплоит за тебя.

## Что такое Replit Agent 3

**Agent 3** — главная фича 2025-2026. Это полноценный AI агент, который:

- Понимает задачу на естественном языке
- Пишет фронтенд и бэкенд самостоятельно
- Дебажит и исправляет ошибки без твоего участия
- Запускает тесты и проверяет edge cases
- Деплоит готовое приложение

Реальный тест: человек без опыта программирования описал задачу — и получил рабочий веб-скрепер за 20 минут.

### Режимы работы агента

**Standard mode** — обычные задачи: создать компонент, добавить endpoint, исправить баг.

**Extended thinking mode** — для сложных архитектурных решений. Агент думает дольше, но выдаёт более продуманное решение.

**High power model** — максимальная точность для критичных задач.

**Web search** — агент сам ищет актуальную документацию и примеры во время работы.

## Регистрация и начало работы

1. Иди на [replit.com](https://replit.com)
2. Зарегистрируйся через GitHub (рекомендую — сразу связываешь аккаунты)
3. Создай новый Repl — выбери шаблон или начни с нуля
4. Опиши что хочешь построить в чате с агентом

```
Пример промпта для агента:
"Создай простое React приложение — список задач с возможностью
добавлять, удалять и отмечать задачи выполненными. 
Используй localStorage для сохранения. Сделай красивый UI с Tailwind."
```

## GitHub интеграция

Replit синхронизируется с GitHub — все изменения автоматически пушатся в репозиторий. Это значит:

- Твой код принадлежит тебе, не Replit
- Можно продолжить работу в любом другом редакторе
- История всех изменений в Git

**Важно:** синхронизация работает с задержкой 30-60 секунд. При быстрой работе могут быть merge conflicts — это нормально.

### Как подключить GitHub репозиторий

1. В настройках Repl → **Git** → **Connect to GitHub repo**
2. Авторизуй Replit в GitHub
3. Выбери существующий репо или создай новый
4. Теперь каждый Push в Replit = коммит в GitHub

## Деплой

Деплой — в один клик прямо из интерфейса.

**Что получаешь:**
- Публичный URL сразу (формат `имя.replit.app`)
- Autoscale инфраструктура через партнёрство с Azure
- Поддержка кастомных доменов
- SSL автоматически

```bash
# Или через Replit CLI (для автоматизации)
replit deploy --project my-app
```

**Для production:** Replit работает как хостинг, но для серьёзных нагрузок лучше деплоить на Railway/Vercel через GitHub интеграцию.

## Базы данных

Replit предоставляет встроенный PostgreSQL. Агент сам настраивает базу когда ты описываешь задачу с данными.

```python
# Пример: агент создаёт такой код автоматически
import os
import psycopg2

conn = psycopg2.connect(os.environ["DATABASE_URL"])
```

Также поддерживаются: SQLite (для простых проектов), Key-Value store (Replit DB).

## Когда использовать Replit

**Replit — лучший выбор когда:**
- Нужно быстро построить прототип с нуля
- Работаешь с несколькими языками в одном проекте
- Важна командная работа в браузере
- Нет желания настраивать локальное окружение

**Лучше взять что-то другое когда:**
- Работаешь над большим production проектом (возьми Cursor)
- Нужен сложный CI/CD (возьми GitHub Actions)
- Критично offline-использование

## Практический пример

Построим простое приложение за 5 минут:

1. Создай новый Repl → "Node.js"
2. Открой Agent chat и напиши:

```
Создай Express API с одним endpoint GET /api/joke,
который возвращает случайный анекдот из массива 10 шуток.
Добавь простую HTML страницу с кнопкой "Получить анекдот".
```

3. Агент напишет сервер, фронтенд и запустит приложение
4. Нажми "Deploy" — получишь публичный URL

Весь процесс занимает 3-5 минут. Без написания единой строки кода вручную.
