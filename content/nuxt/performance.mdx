## ⚡ Производительность Nuxt 3

Nuxt 3 предоставляет широкий набор инструментов для оптимизации производительности: от автоматической оптимизации бандла до edge-деплоя ближе к пользователю.

---

## 📦 Payload Extraction

`payloadExtraction` позволяет отделить данные страницы от JavaScript-бандла:

```typescript
// nuxt.config.ts
export default defineNuxtConfig({
  experimental: {
    payloadExtraction: true
  }
})
```

При статической генерации Nuxt создаёт отдельный `_payload.json` для каждой страницы:

```
dist/
├── _nuxt/          # JS/CSS бандлы
├── about/
│   ├── index.html
│   └── _payload.json  # данные страницы
└── blog/
    ├── my-post/
    │   ├── index.html
    │   └── _payload.json
```

**Преимущества:**
- Меньший начальный HTML
- Данные загружаются параллельно со скриптами
- Лучший кеш для повторных посещений

## 🔗 Preloading и Prefetching с NuxtLink

```vue
<!-- Prefetch при наведении (по умолчанию) -->
<NuxtLink to="/about" prefetch>О нас</NuxtLink>

<!-- Немедленный prefetch при монтировании -->
<NuxtLink to="/dashboard" prefetch prefetch-on="mount">
  Дашборд
</NuxtLink>

<!-- Prefetch только при видимости в viewport -->
<NuxtLink to="/products" prefetch prefetch-on="visible">
  Продукты
</NuxtLink>

<!-- Без prefetch -->
<NuxtLink to="/admin" :prefetch="false">
  Администрирование
</NuxtLink>
```

```typescript
// Глобальная настройка prefetch
// nuxt.config.ts
export default defineNuxtConfig({
  experimental: {
    defaults: {
      nuxtLink: {
        prefetch: true,
        prefetchOn: {
          visibility: true,
          interaction: true
        }
      }
    }
  }
})
```

## 💾 Кеширование

### Route Rules (Nitro)

```typescript
// nuxt.config.ts
export default defineNuxtConfig({
  routeRules: {
    // Статический HTML (SGS)
    '/': { prerender: true },

    // Кеш на CDN 1 час
    '/blog/**': { swr: 3600 },

    // Кеш на 1 день + ISR
    '/docs/**': { isr: 86400 },

    // Без кеша (всегда SSR)
    '/dashboard/**': { ssr: true },

    // Только клиент (SPA)
    '/app/**': { ssr: false },

    // Редирект
    '/old-path': { redirect: '/new-path' }
  }
})
```

### Кеширование API-ответов

```typescript
// server/api/popular-posts.ts
export default defineCachedEventHandler(async (event) => {
  const posts = await fetchPopularPosts()
  return posts
}, {
  maxAge: 60 * 10, // 10 минут
  name: 'popular-posts',
  getKey: () => 'popular-posts'
})
```

## 🚀 Итоги

Производительность Nuxt 3 достигается через:

- **routeRules** — гибкое кеширование: SSR, SSG, ISR, SWR по маршрутам
- **NuxtLink prefetch** — предзагрузка страниц до клика
- **payloadExtraction** — отдельный кеш для данных страниц
- **Edge Deployment** — Nitro на Cloudflare/Vercel Edge
- **@nuxt/image** — автоматическая оптимизация медиа
- **Nuxt DevTools** — профилирование прямо в браузере
