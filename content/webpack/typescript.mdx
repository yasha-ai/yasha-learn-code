# TypeScript в Webpack — два подхода к транспиляции

Webpack поддерживает TypeScript через два основных подхода: **ts-loader** (официальный) и **babel-loader** с `@babel/preset-typescript`. Каждый имеет свои преимущества, и выбор зависит от требований проекта.

## Подход 1: ts-loader

`ts-loader` вызывает TypeScript compiler (`tsc`) напрямую:

```bash
npm install --save-dev typescript ts-loader
```

```js
// webpack.config.js
module: {
  rules: [{
    test: /\\.tsx?$/,
    use: 'ts-loader',
    exclude: /node_modules/,
  }],
},
resolve: {
  extensions: ['.tsx', '.ts', '.js'],
},
```

### Конфигурация tsconfig.json

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "strict": true,
    "jsx": "react-jsx",
    "outDir": "./dist",
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"],
      "@components/*": ["src/components/*"]
    },
    "skipLibCheck": true,
    "esModuleInterop": true,
    "allowSyntheticDefaultImports": true
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist"]
}
```

### ts-loader: transpileOnly режим

По умолчанию ts-loader выполняет полную проверку типов при каждой компиляции — это медленно. Для разработки используйте `transpileOnly`:

```js
{
  loader: 'ts-loader',
  options: {
    transpileOnly: true,     // Только транспиляция, без проверки типов
    happyPackMode: true,     // Для parallel-webpack
    experimentalWatchApi: true,
  },
}
```

Проверку типов перенесите в отдельный процесс:
```bash
npm install --save-dev fork-ts-checker-webpack-plugin
```

```js
const ForkTsCheckerWebpackPlugin = require('fork-ts-checker-webpack-plugin');

plugins: [
  new ForkTsCheckerWebpackPlugin({
    async: true, // Не блокировать сборку
    typescript: {
      diagnosticOptions: {
        semantic: true,
        syntactic: true,
      },
    },
  }),
]
```

## Подход 2: babel-loader + @babel/preset-typescript

Babel просто **удаляет** TypeScript типы без проверки (только транспиляция):

```bash
npm install --save-dev @babel/preset-typescript babel-loader @babel/core @babel/preset-env
```

```js
// webpack.config.js
{
  test: /\\.tsx?$/,
  use: {
    loader: 'babel-loader',
    options: {
      presets: [
        '@babel/preset-env',
        '@babel/preset-react',
        '@babel/preset-typescript',
      ],
    },
  },
  exclude: /node_modules/,
}
```

### Ограничения babel + TypeScript

Babel не поддерживает некоторые TypeScript возможности:
- `const enum` — используйте обычный `enum` или `declare const enum`
- `namespace` — используйте `module`
- `export =` / `import =` — используйте ES импорты
- Декораторы (experimental) — нужен `@babel/plugin-proposal-decorators`

## Сравнение подходов

| Характеристика | ts-loader | babel-loader + TS |
|----------------|-----------|-------------------|
| Проверка типов | ✅ Полная | ❌ Отсутствует (нужен fork-ts-checker) |
| Скорость | Медленнее | Быстрее |
| Конфигурация | Проще | Гибче |
| Декораторы | Нативная поддержка | Нужен плагин |
| `const enum` | ✅ | ❌ |
| React JSX | Нужен `tsx` | Нужен `preset-react` |

## Path Aliases в Webpack

Если в `tsconfig.json` настроены `paths`, нужно дублировать в `resolve.alias`:

```js
// tsconfig.json
{
  "paths": {
    "@/*": ["src/*"],
    "@api/*": ["src/api/*"]
  }
}

// webpack.config.js
const path = require('path');

resolve: {
  alias: {
    '@': path.resolve(__dirname, 'src'),
    '@api': path.resolve(__dirname, 'src/api'),
  },
}
```

Или использовать `tsconfig-paths-webpack-plugin`:
```bash
npm install --save-dev tsconfig-paths-webpack-plugin
```

```js
const TsconfigPathsPlugin = require('tsconfig-paths-webpack-plugin');

resolve: {
  plugins: [new TsconfigPathsPlugin()],
}
// Автоматически читает paths из tsconfig.json!
```

## Declaration files для Webpack

```ts
// typings/declarations.d.ts — типы для нетипизированных ресурсов
declare module '*.svg' { const content: string; export default content; }
declare module '*.png' { const content: string; export default content; }
declare module '*.css' { const styles: Record<string, string>; export default styles; }
declare module '*.module.css' { const styles: Record<string, string>; export default styles; }
```

function generateTsConfig(cfg: Config): string {
  const lines = ['{', '  "compilerOptions": {'];
  lines.push(\`    "target": "\${cfg.target}",\`);
  lines.push('    "module": "ESNext",');
  lines.push('    "moduleResolution": "bundler",');
  if (cfg.strictMode) lines.push('    "strict": true,');
  if (cfg.jsx) lines.push('    "jsx": "react-jsx",');
  lines.push('    "esModuleInterop": true,');
  lines.push('    "skipLibCheck": true,');
  if (cfg.pathAliases) {
    lines.push('    "baseUrl": ".",');
    lines.push('    "paths": {');
    lines.push('      "@/*": ["src/*"]');
    lines.push('    }');
  }
  lines.push('  },');
  lines.push('  "include": ["src/**/*"],');
  lines.push('  "exclude": ["node_modules", "dist"]');
  lines.push('}');
  return lines.join('\\n');
}

function generatePackageInstall(cfg: Config): string {
  const pkgs = ['typescript'];
  if (cfg.approach === 'ts-loader') {
    pkgs.push('ts-loader');
  } else {
    pkgs.push('babel-loader', '@babel/core', '@babel/preset-env', '@babel/preset-typescript');
    if (cfg.jsx) pkgs.push('@babel/preset-react');
  }
  if (cfg.forkTsChecker) pkgs.push('fork-ts-checker-webpack-plugin');
  if (cfg.tsconfigPaths) pkgs.push('tsconfig-paths-webpack-plugin');
  return 'npm install --save-dev ' + pkgs.join(' ');
}
