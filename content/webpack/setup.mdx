# Установка Webpack — первые шаги и базовая конфигурация

Настройка Webpack с нуля — важный навык для любого фронтенд-разработчика. Понимание каждого шага установки помогает разобраться в том, как работает сборщик, и избежать типичных ошибок в реальных проектах.

## Шаг 1 — Инициализация проекта

```bash
mkdir my-webpack-app
cd my-webpack-app
npm init -y
```

После этого в папке появится `package.json`. Флаг `-y` отвечает на все вопросы утвердительно.

## Шаг 2 — Установка Webpack

```bash
# Основные пакеты (всегда в devDependencies)
npm install --save-dev webpack webpack-cli

# Проверка версий
npx webpack --version
# webpack: 5.x.x
# webpack-cli: 5.x.x
```

**Важно**: Webpack устанавливается только в `devDependencies` — он нужен только для сборки, не для runtime.

## Шаг 3 — Структура проекта

```
my-webpack-app/
├── src/
│   ├── index.js       ← точка входа
│   └── utils.js
├── public/
│   └── index.html     ← HTML шаблон
├── dist/              ← сюда Webpack пишет бандлы (создаётся автоматически)
├── webpack.config.js  ← конфиг Webpack
└── package.json
```

## Шаг 4 — webpack.config.js

```js
const path = require('path');
const HtmlWebpackPlugin = require('html-webpack-plugin');

module.exports = {
  // Режим сборки
  mode: 'development',

  // Точка входа
  entry: './src/index.js',

  // Настройки вывода
  output: {
    path: path.resolve(__dirname, 'dist'),
    filename: 'bundle.js',
    clean: true, // Очищать dist перед каждой сборкой (Webpack 5)
  },

  // Плагины
  plugins: [
    new HtmlWebpackPlugin({
      template: './public/index.html',
    }),
  ],

  // Source maps для удобной отладки
  devtool: 'eval-source-map',
};
```

## Шаг 5 — npm scripts

```json
{
  "scripts": {
    "build": "webpack --config webpack.config.js",
    "build:prod": "webpack --config webpack.config.js --mode production",
    "watch": "webpack --watch",
    "dev": "webpack serve --open"
  }
}
```

Теперь запустить сборку можно командой:

```bash
npm run build
```

## Что значит `path.resolve(__dirname, 'dist')`?

- `__dirname` — абсолютный путь до директории текущего файла (webpack.config.js)
- `path.resolve()` — создаёт абсолютный путь, объединяя сегменты
- Результат: `/home/user/projects/my-app/dist`

Webpack всегда ожидает **абсолютные пути** для `output.path` — именно поэтому нельзя написать просто `'./dist'`.

## Опциональные пакеты для старта

```bash
# Dev сервер с HMR
npm install --save-dev webpack-dev-server

# HTML плагин
npm install --save-dev html-webpack-plugin

# Babel для ES6+ и React
npm install --save-dev babel-loader @babel/core @babel/preset-env @babel/preset-react

# TypeScript
npm install --save-dev typescript ts-loader
```

function generateConfig(enabled: Set<string>, mode: string): string {
  const lines: string[] = ["const path = require('path');"];

  if (enabled.has('html')) lines.push("const HtmlWebpackPlugin = require('html-webpack-plugin');");
  if (enabled.has('miniCss')) lines.push("const MiniCssExtractPlugin = require('mini-css-extract-plugin');");

  lines.push('', 'module.exports = {');
  lines.push(\`  mode: '\${mode}',\`);
  lines.push("  entry: './src/index" + (enabled.has('typescript') ? '.ts' : '.js') + "',");
  lines.push('  output: {');
  lines.push("    path: path.resolve(__dirname, 'dist'),");
  lines.push("    filename: '[name].[contenthash].js',");

  lines.push('  },');

    lines.push('  resolve: {');
    lines.push("    extensions: ['.tsx', '.ts', '.js'],");
    lines.push('    alias: {');
    lines.push("      '@': path.resolve(__dirname, 'src'),");
    lines.push('    },');
    lines.push('  },');
  }

  if (enabled.has('sourceMap')) {
    lines.push(\`  devtool: '\${mode === 'development' ? 'eval-source-map' : 'source-map'}',\`);
  }

  const rules: string[] = [];
  if (enabled.has('typescript') && enabled.has('babel')) {
    rules.push("    { test: /\\\\.(ts|tsx)$/, use: ['babel-loader', 'ts-loader'], exclude: /node_modules/ },");
  } else if (enabled.has('typescript')) {
    rules.push("    { test: /\\\\.(ts|tsx)$/, use: 'ts-loader', exclude: /node_modules/ },");
  } else if (enabled.has('babel')) {
    rules.push("    { test: /\\\\.(js|jsx)$/, use: 'babel-loader', exclude: /node_modules/ },");
  }
  if (enabled.has('sass') && enabled.has('css')) {
    const cssUse = enabled.has('miniCss') ? 'MiniCssExtractPlugin.loader' : "'style-loader'";
    rules.push(\`    { test: /\\\\.s[ac]ss$/i, use: [\${cssUse}, 'css-loader', 'sass-loader'] },\`);
  } else if (enabled.has('css')) {
    const cssUse = enabled.has('miniCss') ? 'MiniCssExtractPlugin.loader' : "'style-loader'";
    rules.push(\`    { test: /\\\\.css$/, use: [\${cssUse}, 'css-loader'] },\`);
  }
  if (rules.length > 0) {
    lines.push('  module: {');
    lines.push('    rules: [');
    rules.forEach(r => lines.push('  ' + r));
    lines.push('    ],');
    lines.push('  },');
  }

  const plugins: string[] = [];
  if (enabled.has('html')) plugins.push("    new HtmlWebpackPlugin({ template: './public/index.html' }),");
  if (enabled.has('miniCss')) plugins.push("    new MiniCssExtractPlugin({ filename: '[name].[contenthash].css' }),");
  if (plugins.length > 0) {
    lines.push('  plugins: [');
    plugins.forEach(p => lines.push('  ' + p));
    lines.push('  ],');
  }

  if (enabled.has('devServer')) {
    lines.push('  devServer: {');
    lines.push("    static: './dist',");
    lines.push('    hot: true,');
    lines.push('    port: 3000,');
    if (enabled.has('html')) lines.push('    historyApiFallback: true,');
    lines.push('  },');
  }

  if (enabled.has('splitChunks')) {
    lines.push('  optimization: {');
    lines.push('    splitChunks: {');
    lines.push("      chunks: 'all',");
    lines.push('    },');
    if (mode === 'production') lines.push('    runtimeChunk: true,');
    lines.push('  },');
  }

  lines.push('};');
  return lines.join('\\n');
}
