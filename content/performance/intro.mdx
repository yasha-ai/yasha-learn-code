# Введение в Performance & SEO

Производительность и SEO — это не просто "ускорить сайт". Это деньги, пользователи и позиции в Google.

**Почему это важно?**
- Amazon: +100ms = -1% продаж
- Google: скорость = фактор ранжирования
- Пользователи: 53% уходят если загрузка > 3 сек

## Что такое Web Performance?

Web Performance — это **скорость, с которой страница загружается, отображается и реагирует** на действия пользователя.

Делится на 3 части:
- **Loading** — скачивание ресурсов
- **Rendering** — отрисовка страницы
- **Interaction** — реакция на действия

## Что такое SEO?

SEO (Search Engine Optimization) — это **оптимизация сайта для поисковых систем**.

Основные направления:
- **On-page** — контент, теги, структура
- **Technical** — скорость, индексация, структура URL
- **Off-page** — ссылки, авторитет домена

## Связь Performance и SEO

С 2021 года Google использует **Core Web Vitals** как фактор ранжирования.  
Медленный сайт = низкие позиции = меньше трафика.

```
Производительность → UX → SEO
```

## Инструменты

| Инструмент | Назначение |
|---|---|
| Lighthouse | Аудит производительности и SEO |
| Chrome DevTools | Профилирование в реальном времени |
| PageSpeed Insights | Оценка + рекомендации Google |
| WebPageTest | Детальный анализ загрузки |
| GTmetrix | Комплексная аналитика |

## Ключевые метрики

```
LCP  — Largest Contentful Paint    → < 2.5 сек
FID  — First Input Delay           → < 100 мс
CLS  — Cumulative Layout Shift     → < 0.1
INP  — Interaction to Next Paint   → < 200 мс
TTFB — Time to First Byte          → < 800 мс
FCP  — First Contentful Paint      → < 1.8 сек
```

## Практические задания

1. Откройте Chrome DevTools → Network и посмотрите как загружается любой сайт
2. Запустите Lighthouse аудит на вашем проекте (F12 → Lighthouse)
3. Проверьте PageSpeed Insights для `https://google.com`
4. Найдите 3 проблемы производительности на любом сайте
