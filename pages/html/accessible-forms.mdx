# HTML: Доступные формы

В этом уроке мы рассмотрим, как создавать доступные формы. Это критически важно для того, чтобы ваш сайт был удобен всем пользователям, включая тех, кто использует скринридеры или навигацию с клавиатуры.

### Основные принципы

* **Использование `<label>`:** Тег `<label>` связывает текст с полем ввода. Это позволяет скринридеру озвучить название поля при фокусе.

<Playground
  template="static"
  files={{
    "index.html": `<label for="name">Ваше имя:</label>
<input type="text" id="name" name="name" placeholder="Иван">`
  }}
/>

* **ARIA-атрибуты:** Если визуальной подписи нет (например, только иконка поиска), используйте `aria-label`.

<Playground
  template="static"
  files={{
    "index.html": `<input type="text" aria-label="Поиск по сайту" placeholder="Найти...">`
  }}
/>

* **Обработка ошибок:** Используйте `aria-invalid` и `aria-describedby` для связи поля с текстом ошибки.

<Playground
  template="static"
  files={{
    "index.html": `<div>
  <label for="pass">Пароль:</label>
  <input type="password" id="pass" aria-invalid="true" aria-describedby="pass-error">
  <p id="pass-error" style="color: red;">Пароль должен быть длиннее 8 символов.</p>
</div>`
  }}
/>

* **Группировка полей:** Используйте `<fieldset>` и `<legend>` для логически связанных полей.

<Playground
  template="static"
  files={{
    "index.html": `<fieldset style="padding: 15px; border-radius: 8px;">
  <legend>Контактные данные</legend>
  <label for="email">Email:</label><br>
  <input type="email" id="email"><br><br>
  <label for="phone">Телефон:</label><br>
  <input type="tel" id="phone">
</fieldset>`
  }}
/>

### Жизненный пример

Государственные сайты и крупные сервисы (Google, Apple) строго следуют этим правилам. Библиотеки вроде **React Aria** помогают разработчикам не забывать об этих атрибутах, управляя ими автоматически.

### Ключевые моменты

* Всегда связывайте `<label>` и `input` через `for` и `id`.
* Сообщения об ошибках должны быть программно связаны с полем.
* Не полагайтесь только на `placeholder` — он исчезает при вводе и часто не озвучивается как метка.
* Группируйте сложные формы через `fieldset`.
* Тестируйте формы, используя только клавиатуру.
