# HTML: Основы ARIA

В этом уроке мы познакомимся с ARIA – набором атрибутов, которые делают веб-контент более доступным для людей с ограниченными возможностями. ARIA расширяет HTML, позволяя нам описывать роли, состояния и свойства элементов для скринридеров.

### Что такое ARIA?

ARIA (Accessible Rich Internet Applications) – это набор атрибутов, которые предоставляют дополнительную информацию о роли, состоянии и свойствах элементов интерфейса. Если стандартный HTML-тег не может передать смысл элемента, ARIA приходит на помощь.

### Основные типы атрибутов

* **role**: Определяет роль элемента (например, `button`, `menu`, `alert`).
* **aria-state**: Описывает текущее состояние (например, `aria-expanded="true"` — развернуто).
* **aria-property**: Определяет свойства (например, `aria-label="Закрыть"` — текстовая метка).

### Практические примеры

**Пример 1: Кнопка-иконка**
Если на кнопке только крестик, скринридер может прочитать его как "икс". С помощью `aria-label` мы даем кнопке понятное описание.

<Playground
  template="static"
  files={{
    "index.html": `<button aria-label="Закрыть модальное окно" style="padding: 10px; cursor: pointer;">
  X
</button>`
  }}
/>

**Пример 2: Сообщение об ошибке**
Атрибут `aria-describedby` связывает поле ввода с текстом описания или ошибки.

<Playground
  template="static"
  files={{
    "index.html": `<div>
  <label for="email">Email:</label>
  <input type="email" id="email" aria-describedby="email-error">
  <p id="email-error" style="color: red;">Пожалуйста, введите корректный адрес почты.</p>
</div>`
  }}
/>

**Пример 3: Кастомный выпадающий список**
Когда мы создаем сложные элементы из обычных `div`, ARIA помогает скринридеру понять их структуру.

<Playground
  template="static"
  files={{
    "index.html": `<div role="combobox" aria-expanded="true" aria-haspopup="listbox">
  <label>Выберите вариант:</label>
  <ul role="listbox" style="border: 1px solid #ccc; padding: 10px; list-style: none;">
    <li role="option">Вариант 1</li>
    <li role="option">Вариант 2</li>
  </ul>
</div>`
  }}
/>

### Ключевые моменты

* ARIA улучшает доступность для людей, использующих вспомогательные технологии.
* Используйте ARIA только тогда, когда семантики обычного HTML недостаточно.
* Правило №1 в ARIA: если можно использовать нативный HTML-тег (например, `<button>` вместо `<div role="button">`), используйте его.
* Всегда тестируйте доступность интерфейса.
