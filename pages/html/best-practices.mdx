# HTML: Лучшие практики

Привет! Яша снова на связи. Сегодня мы погрузимся в мир "лучших практик" (best practices). Это правила, которые делают ваш код понятным, быстрым и удобным для всех.

### Почему это важно?

* **Доступность (a11y):** Чтобы сайтом могли пользоваться все.
* **Производительность:** Быстрая загрузка радует и людей, и поисковики.
* **SEO:** Поисковые системы лучше понимают ваш контент.
* **Поддержка:** Чистый код легче изменять вам и другим разработчикам.

### Основные принципы

#### 1. Семантический HTML
Используйте теги по смыслу. Избегайте "div-супа"!

<Playground
  template="static"
  files={{
    "index.html": `<header>
  <h1>Yasha Learn Code</h1>

  </nav>
</header>

<main>
  <article>
    <h2>Как писать чистый код</h2>
    <p>Используйте семантические теги...</p>
  </article>
</main>

<footer>
  <p>&copy; 2026 Яша Учит Код</p>
</footer>`
  }}
/>

#### 2. Доступность
Не забывайте про `alt` для картинок и `label` для форм.

<Playground
  template="static"
  files={{
    "index.html": `<img 
  src="https://images.unsplash.com/photo-1587620962725-abab7fe55159?w=300" 
  alt="Код на экране ноутбука"
>

<div style="margin-top: 20px;">
  <label for="username">Имя пользователя:</label>
  <input type="text" id="username" name="username">
</div>`
  }}
/>

#### 3. Структура форм
Группируйте связанные поля.

<Playground
  template="static"
  files={{
    "index.html": `<form>
  <fieldset style="padding: 15px; border-radius: 8px;">
    <legend>Контактные данные</legend>
    <label for="email">Email:</label>
    <input type="email" id="email" required>
  </fieldset>
  <button type="submit" style="margin-top: 10px;">Отправить</button>
</form>`
  }}
/>

#### 4. Оптимизация изображений
Указывайте размеры и используйте отложенную загрузку (`loading="lazy"`).

<Playground
  template="static"
  files={{
    "index.html": `<img 
  src="https://images.unsplash.com/photo-1498050108023-c5249f4df085?w=1200" 
  alt="Рабочее место разработчика" 
  width="1200" 
  height="600" 
  loading="lazy"
>`
  }}
/>

### Ключевые моменты

* Используйте семантические теги вместо `div` там, где это возможно.
* Всегда добавляйте атрибут `alt` к изображениям.
* Связывайте `label` и `input` через `id`.
* Добавляйте `loading="lazy"` для картинок вне первого экрана.
* Проверяйте код через валидатор W3C.
