# HTML: Скринридеры


![Иллюстрация к уроку](/lessons/html-screen-readers.png)
Screen Readers (программы чтения с экрана) – это технологии, позволяющие людям с нарушениями зрения получать доступ к цифровому контенту. Они читают текст, описывают изображения и сообщают о структуре страницы. В этом уроке мы рассмотрим, как сделать ваш HTML-код понятным для скринридеров.

## Как скринридеры работают с HTML

Скринридеры анализируют HTML-код, следуя логической структуре документа. Важно использовать семантические теги и атрибуты, чтобы обеспечить правильную интерпретацию контента.

## Примеры кода

### Семантические теги

Вместо того чтобы делать всё на `<div>`, используйте теги, описывающие структуру.

<Playground
  template="static"
  files={{
    "index.html": `<header>
  <h1>Заголовок сайта</h1>
  <nav>
    <a href="#">Главная</a> | <a href="#">О нас</a>
  </nav>
</header>

<main>
  <article>
    <h2>Заголовок статьи</h2>
    <p>Текст статьи, который будет прочитан скринридером.</p>
  </article>
</main>

<footer>
  <p>&copy; 2026 Мой сайт</p>
</footer>`
  }}
/>

### Атрибут alt для изображений

Обязательно добавляйте `alt`. Он описывает изображение для тех, кто его не видит.

<Playground
  template="static"
  files={{
    "index.html": `<!-- Информативное изображение -->
<img src="https://images.unsplash.com/photo-1546182990-dffeafbe841d?w=200" alt="Лев в саванне">

<!-- Декоративное изображение (скринридер его пропустит) -->
<img src="https://www.w3.org/WAI/alt/img/border.png" alt="">`
  }}
/>

### Атрибуты aria-*

Они дают дополнительную информацию. Например, `aria-current="page"` сообщает, что ссылка ведет на текущую страницу.

<Playground
  template="static"
  files={{
    "index.html": `<nav>
  <ul>
    <li><a href="/" aria-current="page">Главная (текущая)</a></li>
    <li><a href="/blog">Блог</a></li>
  </ul>
</nav>`
  }}
/>

### Заголовки

Используйте заголовки (`<h1>` - `<h6>`) по порядку. Скринридеры позволяют пользователям быстро перемещаться между ними.

<Playground
  template="static"
  files={{
    "index.html": `<h1>Заголовок уровня 1</h1>
<section>
  <h2>Заголовок уровня 2</h2>
  <p>Контент раздела.</p>
</section>`
  }}
/>

## Ключевые моменты

* Используйте семантические теги (`main`, `nav`, `article`).
* Всегда добавляйте `alt` к изображениям.
* Используйте `aria-*` для сложных интерфейсов.
* Сохраняйте логический порядок заголовков.
* Тестируйте сайт с реальными скринридерами (например, NVDA или VoiceOver).
