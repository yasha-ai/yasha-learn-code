# HTML: ARIA Roles


![Иллюстрация к уроку](/lessons/html-aria-roles.png)
ARIA Roles – это важная часть обеспечения доступности веб-страниц для людей с ограниченными возможностями. Они помогают сделать контент более понятным для вспомогательных технологий, таких как скринридеры.

## Что такое ARIA Roles?

ARIA (Accessible Rich Internet Applications) – это набор атрибутов HTML, которые улучшают доступность веб-контента. Атрибут `role` определяет, чем является элемент на странице. Он сообщает скринридеру функцию элемента, особенно если стандартный HTML-тег не передает нужный смысл.

Представьте, что у вас есть `div`, который ведет себя как кнопка. Сам по себе `div` не имеет семантического значения кнопки. ARIA `role` позволяет это исправить.

### Пример: Кнопка из Div

<Playground
  template="static"
  files={{
    "index.html": `<div 
  onclick="alert('Нажато!')" 
  role="button" 
  tabindex="0" 
  style="display: inline-block; padding: 10px; background: #3498db; color: white; border-radius: 4px; cursor: pointer;"
>
  Нажми меня
</div>`
  }}
/>

В этом примере мы говорим скринридеру, что этот `div` – кнопка. `tabindex="0"` делает элемент доступным для навигации с помощью клавиши Tab.

## Часто используемые роли

* `button`: Кнопка.
* `link`: Ссылка.
* `navigation`: Блок навигации.
* `main`: Основное содержание страницы.
* `article`: Независимый блок контента (например, статья).
* `alert`: Важное сообщение, требующее немедленного внимания.

### Пример: Alert (Уведомление)

## Жизненный пример

Многие современные JavaScript-фреймворки (React, Angular, Vue) используют ARIA roles для создания сложных компонентов, таких как вкладки (tabs), модальные окна или выпадающие меню. Это позволяет пользователям скринридеров понимать структуру приложения и успешно взаимодействовать с ним.

## Ключевые моменты

* ARIA roles добавляют семантику там, где её не хватает в стандартном HTML.
* Используйте `role`, чтобы сделать сайт доступным.
* Проверяйте доступность вашего сайта с помощью скринридеров.
* Правило №1: используйте нативные теги (`<button>`, `<nav>`), если это возможно. Используйте ARIA только как дополнение.
