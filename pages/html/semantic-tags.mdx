# HTML: Семантические теги

![Иллюстрация к уроку](/lessons/html-semantic-tags.png)

Семантические теги в HTML помогают сделать структуру вашей веб-страницы более понятной и осмысленной как для браузеров, так и для разработчиков. Они несут информацию о *значении* контента, а не только о его внешнем виде.

## Что такое семантические теги?

Семантические теги — это HTML-элементы, которые описывают *тип* контента, который они содержат. В отличие от `<div>` и `<span>`, которые являются общими контейнерами, семантические теги несут конкретный смысл. Например, тег `<article>` предназначен для статей, а `<nav>` — для навигационных элементов.

**Примеры:**

* `<article>`: Статья, блог-пост, новость.
* `<aside>`: Дополнительная информация, боковая панель.
* `<nav>`: Навигационное меню.
* `<header>`: Заголовок раздела или страницы.
* `<footer>`: Подвал раздела или страницы.
* `<main>`: Основное содержимое страницы.
* `<section>`: Раздел контента.

## Практические примеры

Рассмотрим пример структуры веб-страницы с использованием семантических тегов:

<Playground
  template="static"

    "index.html": `<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">

    <title>Семантические теги</title>

        <h1>Мой блог</h1>

            <ul>
                <li><a href="#">Главная</a></li>
                <li><a href="#">Обо мне</a></li>
                <li><a href="#">Контакты</a></li>
            </ul>
        </nav>
    </header>

    <main>
        <article>
            <h2>Заголовок статьи</h2>
            <p>Текст статьи...</p>
        </article>

        <aside>
            <h3>Полезные ссылки</h3>
            <ul>
                <li><a href="#">Ссылка 1</a></li>
                <li><a href="#">Ссылка 2</a></li>
            </ul>
        </aside>
    </main>

    <footer>
        <p>© 2023 Мой блог</p>
    </footer>

</body>
</html>`
  }}
/>

В этом примере:

* `<header>` содержит заголовок страницы и навигационное меню.
* `<nav>` определяет навигационный блок.
* `<main>` содержит основное содержимое страницы.
* `<article>` представляет собой отдельную статью.
* `<aside>` содержит дополнительную информацию (боковую панель).
* `<footer>` отображает информацию о подвале страницы.

## Жизненный пример

Семантические теги активно используются в современных веб-разработках. Например, в React, Vue и Angular при создании компонентов часто используются семантические теги для структурирования разметки. На многих новостных сайтах вы увидите использование `<article>`, `<aside>`, `<nav>`, `<header>` и `<footer>` для четкой организации контента. Использование семантики улучшает SEO (поисковую оптимизацию), так как поисковые системы лучше понимают структуру вашего сайта.

## Ключевые моменты

* Семантические теги описывают *значение* контента, а не только его внешний вид.
* Использование семантических тегов делает код более понятным и читаемым.
* Семантика улучшает SEO.
* Примеры семантических тегов: `<article>`, `<aside>`, `<nav>`, `<header>`, `<footer>`, `<main>`, `<section>`.
* Не забывайте использовать `<main>` для обозначения основного контента страницы.
* Используйте `<div>` только тогда, когда нет более подходящего семантического тега.
