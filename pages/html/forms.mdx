## HTML: Формы и инпуты

### 🚀 Что это и зачем

Формы — это сердце интерактивности на любом сайте! Они позволяют пользователям вводить данные: регистрироваться, входить в аккаунт, отправлять сообщения, искать информацию или делать заказы. Без форм ты не сможешь получить от пользователя ни одного байта информации. Все, что ты вводишь в поля поиска, логина или комментария — это части HTML-форм.

Когда ты нажимаешь кнопку "Отправить", данные из формы путешествуют на сервер, где их обрабатывает специальная программа (например, на PHP, Python, Node.js).

### 💡 Основная теория с примерами

Основной элемент формы — это тег `<form>`. Внутри него ты размещаешь все контролы (поля ввода, кнопки).

*   `action`: URL-адрес, куда будут отправлены данные формы.
*   `method`: HTTP-метод для отправки данных. Чаще всего `GET` (данные видны в URL) или `POST` (данные отправляются "в фоне", безопаснее для чувствительной информации).

Самый универсальный элемент формы — это `<input>`. Его поведение сильно зависит от атрибута `type`.
Важно! Каждый `<input>` должен иметь атрибут `name`, чтобы данные можно было идентифицировать на сервере. Атрибут `id` нужен для связывания с `<label>`.

```html
<!-- Пример 1: Простая форма ввода имени -->
<form action="/submit-name" method="post">
    <!-- label улучшает доступность и удобство использования:
         при клике на текст "Имя:" активируется поле ввода -->
    <label for="userName">Имя:</label>
    <!-- input с типом "text" для обычного текста -->
    <input type="text" id="userName" name="username" placeholder="Введите ваше имя">

    <br> <!-- Перевод строки для наглядности -->

    <label for="userEmail">Email:</label>
    <!-- input с типом "email" автоматически проверяет формат email -->
    <input type="email" id="userEmail" name="email" required placeholder="example@domain.com">

    <br>

    <!-- Кнопка для отправки формы -->
    <button type="submit">Отправить</button>
</form>
```

### ✨ Продвинутые примеры

HTML5 предлагает множество специализированных типов `<input>` для лучшего UX и базовой валидации.

```html
<!-- Пример 2: Форма регистрации с разными типами инпутов -->
<form action="/register" method="post">
    <h3>Регистрация</h3>

    <label for="regUsername">Логин:</label>
    <input type="text" id="regUsername" name="username" required minlength="3" maxlength="20" placeholder="От 3 до 20 символов">
    <br>

    <label for="regPassword">Пароль:</label>
    <!-- type="password" скрывает вводимые символы -->
    <input type="password" id="regPassword" name="password" required minlength="8" placeholder="Минимум 8 символов">
    <br>

    <label for="birthDate">Дата рождения:</label>
    <!-- type="date" открывает календарь для выбора даты -->
    <input type="date" id="birthDate" name="birthdate">
    <br>

    <label for="agreeTerms">Согласен с условиями:</label>
    <!-- type="checkbox" для выбора одного или нескольких вариантов -->
    <input type="checkbox" id="agreeTerms" name="terms" value="agreed" required>
    <br>

    <button type="submit">Зарегистрироваться</button>
</form>
```

Помимо `<input>`, есть и другие важные элементы форм:

```html
<!-- Пример 3: Форма обратной связи с текстовым полем и выпадающим списком -->
<form action="/feedback" method="post">
    <h3>Оставьте отзыв</h3>

    <label for="feedbackType">Тип обращения:</label>
    <!-- select позволяет выбрать один пункт из выпадающего списка -->
    <select id="feedbackType" name="type">
        <option value="question">Вопрос</option>
        <option value="suggestion">Предложение</option>
        <option value="complaint">Жалоба</option>
    </select>
    <br><br>

    <label for="message">Ваше сообщение:</label>
    <!-- textarea для многострочного ввода текста -->
    <textarea id="message" name="message" rows="5" cols="30" placeholder="Напишите здесь ваше сообщение..."></textarea>
    <br>

    <button type="submit">Отправить сообщение</button>
</form>
```

```html
<!-- Пример 4: Выбор пола с помощью radio-кнопок -->
<form action="/set-gender" method="post">
    <h3>Укажите ваш пол:</h3>

    <!-- input type="radio" позволяет выбрать только один вариант из группы.
         Обязательно используй одно и то же name для всех элементов группы. -->
    <input type="radio" id="genderMale" name="gender" value="male">
    <label for="genderMale">Мужской</label><br>

    <input type="radio" id="genderFemale" name="gender" value="female">
    <label for="genderFemale">Женский</label><br>

    <input type="radio" id="genderOther" name="gender" value="other">
    <label for="genderOther">Другой</label><br><br>

    <button type="submit">Сохранить</button>
</form>
```

```html
<!-- Пример 5: Ввод числа с ограничениями -->
<form action="/set-quantity" method="post">
    <label for="quantity">Количество (от 1 до 10):</label>
    <!-- type="number" принимает только числа.
         Атрибуты min, max, step позволяют задать диапазон и шаг изменения. -->
    <input type="number" id="quantity" name="itemQuantity" min="1" max="10" value="1">
    <button type="submit">Заказать</button>
</form>
```

### 💣 Типичные ошибки

1.  **Забывать `name` атрибут**: Без него данные из поля не будут отправлены на сервер.
2.  **Не использовать `<label>`**: Это ухудшает доступность (для скринридеров) и пользовательский опыт (сложнее попасть в маленькое поле).
3.  **Не использовать `for` в `<label>` и `id` в `<input>`**: Без их связки `label` не будет "активировать" поле ввода.

### 🎯 Практика

1.  Создай форму для оформления подписки на рассылку. Форма должна содержать поле для ввода email (обязательное), поле для имени (необязательное) и кнопку "Подписаться".
2.  Создай форму поиска. Одно текстовое поле и кнопка "Поиск".
3.  Разработай простую форму заказа пиццы. Она должна включать:
    *   Выбор размера пиццы (маленькая, средняя, большая) с помощью `radio` кнопок.
    *   Поле для дополнительных комментариев (`textarea`).
    *   Кнопку "Заказать".

### 💡 Совет

Всегда помни, что клиентская валидация (через атрибуты `required`, `minlength`, `type="email"` и т.д.) — это лишь первый барьер. Настоящая и надежная валидация должна происходить на сервере, чтобы защититься от злонамеренных действий или ошибок, которые обошли клиентские проверки.
