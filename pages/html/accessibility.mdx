# HTML: Доступность (a11y)

![Иллюстрация к уроку](/lessons/html-accessibility.png)


Привет! Яша на связи. Сегодня мы погрузимся в очень важную тему — **доступность (accessibility или a11y)**. Это возможность для всех пользователей успешно взаимодействовать с твоим сайтом.

### Почему это важно?

1. **Инклюзивность:** Интернет должен быть открыт для каждого.
2. **Удобство:** Доступные сайты удобнее для всех (например, хороший контраст помогает читать на солнце).
3. **SEO:** Поисковики лучше индексируют семантически верные сайты.

### Основы доступности

Самый первый шаг — использование **семантического HTML**.

#### 1. Семантические теги

Используй теги по назначению. Браузеры и скринридеры используют структуру HTML, чтобы понять содержание.

<Playground
  template="static"
  files={{
    "index.html": `<!-- ХОРОШО: Используем смысловые теги -->
<header>
  <h1>Заголовок страницы</h1>
</header>
<nav>
  <a href="#">Ссылка 1</a> | <a href="#">Ссылка 2</a>
</nav>
<main>
  <p>Основной контент страницы.</p>
</main>`
  }}
/>

#### 2. Атрибут alt для изображений

Скринридеры озвучивают текст из `alt`, помогая понять, что на картинке.

<Playground
  template="static"
  files={{
    "index.html": `<img 
  src="https://images.unsplash.com/photo-1514888286974-6c03e2ca1dba?w=200" 
  alt="Милый рыжий кот спит на диване"
>`
  }}
/>

#### 3. Атрибут label для форм

`label` связывает текст с полем ввода.

<Playground
  template="static"
  files={{
    "index.html": `<label for="userName">Имя пользователя:</label>
<input type="text" id="userName" name="user" placeholder="Введите имя">`
  }}
/>

#### 4. Навигация с клавиатуры

Интерактивные элементы должны быть доступны через клавишу `Tab`.

<Playground
  template="static"
  files={{
    "index.html": `<!-- Кнопка из div требует tabindex и role -->
<div 
  tabindex="0" 
  role="button" 
  onclick="alert('Нажато!')"
  style="display: inline-block; padding: 10px; background: #eee; cursor: pointer;"
>
  Нажми меня (работает с Tab)
</div>`
  }}
/>

### ARIA-атрибуты

ARIA дополняет HTML, предоставляя информацию там, где стандартных тегов недостаточно.

<Playground
  template="static"
  files={{
    "index.html": `<button aria-expanded="false" onclick="this.setAttribute('aria-expanded', 'true')">
  Открыть меню
</button>`
  }}
/>

### Ключевые моменты

* Используйте семантические теги (`h1-h6`, `nav`, `main`, `section`).
* Всегда пишите осмысленный `alt` для картинок.
* Связывайте `label` и `input` через `id`.
* Проверяйте контрастность текста и фона.
* Убедитесь, что сайтом можно пользоваться без мыши.



## Интерактивный пример

<Sandpack
  template="vanilla"
  files={{
    "/index.html": `
<style>
  body { background: #282c34; color: white; font-family: sans-serif; padding: 20px; }
  main { background: #333842; padding: 20px; border-radius: 8px; max-width: 250px; }
  img { width: 100%; border-radius: 4px; margin-bottom: 10px; }
  label { display: block; margin-bottom: 5px; font-size: 14px; }
  input { width: 100%; padding: 8px; margin-bottom: 15px; box-sizing: border-box; }
  .btn { padding: 10px; background: #61dafb; color: #000; cursor: pointer; text-align: center; border-radius: 4px; font-weight: bold; }
  /* Видимый фокус для доступности */
  .btn:focus { outline: 2px solid #fff; outline-offset: 2px; }
</style>

<!-- Семантический тег main -->
<main>
  <!-- Атрибут alt для скринридеров -->
  <img src="https://images.unsplash.com/photo-1514888286974-6c03e2ca1dba?w=200" alt="Милый рыжий кот">
  
  <!-- Связка label и input через for/id -->
  <label for="catName">Имя кота:</label>
  <input type="text" id="catName" placeholder="Например, Барсик">
  
  <!-- Кастомная кнопка с tabindex и role -->
  <div class="btn" role="button" tabindex="0" id="btn">
    Погладить (Tab + Enter)
  </div>
</main>

<script>
  const btn = document.getElementById('btn');
  const action = () => alert('Мур-мур! Кот доволен.');
  
  btn.onclick = action;
  
  // Обработка клавиатуры для кастомной кнопки
  btn.onkeydown = (e) => { 
    if(e.key === 'Enter' || e.key === ' ') { 
      e.preventDefault(); 
      action(); 
    } 
  };
<\/script>
`
  }}
/>
